# Инструкция по использованию CRUD операций

## Что было реализовано

Согласно заданию, реализованы CRUD операции (Create, Read, Update, Delete) для модели **Service** (Услуга) на веб-страницах проекта.

## Доступные страницы

### 1. Главная страница
**URL:** `/`  
**Описание:** Приветственная страница с навигацией

### 2. Список услуг (Read - Просмотр)
**URL:** `/services/`  
**Функционал:**
- Отображение всех услуг в виде таблицы
- Поиск по названию услуги
- Фильтрация по категориям
- Пагинация (10 записей на страницу)
- Кнопки для просмотра, редактирования и удаления каждой услуги

### 3. Детальная информация об услуге (Read - Просмотр)
**URL:** `/services/<id>/`  
**Функционал:**
- Отображение полной информации об услуге
- Кнопки для редактирования и удаления

### 4. Добавление услуги (Create - Создание)
**URL:** `/services/create/`  
**Функционал:**
- Форма для добавления новой услуги
- Поля: название, категория, срок выполнения, описание
- Валидация данных
- Сообщение об успешном добавлении

### 5. Редактирование услуги (Update - Обновление)
**URL:** `/services/<id>/update/`  
**Функционал:**
- Форма для изменения данных услуги
- Предзаполнение полей текущими значениями
- Сообщение об успешном обновлении

### 6. Удаление услуги (Delete - Удаление)
**URL:** `/services/<id>/delete/`  
**Функционал:**
- Страница подтверждения удаления
- Отображение информации об удаляемой услуге
- Сообщение об успешном удалении

## Как протестировать

1. **Запустите сервер:**
   ```bash
   python manage.py runserver
   ```

2. **Откройте браузер и перейдите на:** http://127.0.0.1:8000/

3. **Создайте тестовые данные через админку:**
   - Зайдите в админ-панель: http://127.0.0.1:8000/admin/
   - Создайте несколько категорий услуг (ServiceCategory)
   - Затем можете создавать услуги через веб-интерфейс

4. **Протестируйте CRUD операции:**
   - Перейдите на `/services/`
   - Нажмите "Добавить услугу" (Create)
   - Заполните форму и сохраните
   - Нажмите на название услуги для просмотра (Read)
   - Нажмите "Редактировать" для изменения (Update)
   - Нажмите "Удалить" для удаления (Delete)

## Технические детали

### Использованные технологии Django:
- **Class-Based Views:** ListView, DetailView, CreateView, UpdateView, DeleteView
- **ModelForm:** для работы с формами
- **Django Messages Framework:** для уведомлений пользователя
- **Пагинация:** встроенная пагинация Django
- **URL routing:** использование app_name и namespaces

### Структура файлов:
- `mfc/views.py` - все представления
- `mfc/forms.py` - форма для работы с услугами
- `mfc/urls.py` - маршрутизация приложения
- `mfc/templates/mfc/` - все HTML шаблоны
